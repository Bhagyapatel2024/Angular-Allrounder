Angular Concepts

What is Angular?

Angular is an open-source, TypeScript-based framework developed by Google. It is used to build single-page web applications (SPAs) with features like reusable components, two-way data binding, dependency injection, and routing.

1. Modules

Modules are the building blocks of Angular applications. The root module (`AppModule`) serves as the entry point for the app.

  • Organize the application into cohesive blocks of functionality.
  • Import Angular libraries like `FormsModule` or custom feature modules.
  • Help in lazy loading features for better performance.

2. Components

Components control a section of the UI and define the application's logic and view. A component is made up of:

  • HTML Template: Defines the UI structure.
  • TypeScript Class: Contains the logic and data for the template.
  • CSS Styles: Provides styling for the template.

3. Templates

Templates are written in HTML and define how the component should be rendered on the screen. Angular templates can include:

  • HTML code.
  • Angular directives like `*ngIf`, `*ngFor` for conditional rendering and loops.
  • Data binding to dynamically display or update content.

4. Data Binding

Angular provides data binding to synchronize the view (HTML) and model (TypeScript logic). It can be one-way or two-way:

  • Interpolation: ` data ` for displaying dynamic data.
  • Property Binding: `[property]="data"` to bind properties.
  • Event Binding: `(event)="method()"` to handle events like clicks.
  • Two-Way Binding: `[(ngModel)]="data"` for real-time updates.

5. Services and Dependency Injection

Services are used to share logic or data between different parts of the application. Dependency injection (DI) allows components to access these services efficiently.

  • Inject a service using the component's constructor.
  • Define shared logic like API calls, authentication, etc., in services.

6. Routing

Routing enables navigation between different views or pages in the application. Angular's `RouterModule` is used to define routes.

  • Use router-outlet in the template to render views.
  • Define routes in the `app-routing.module.ts` file.
  • Lazy load modules for better performance.

7. Directives

Directives are used to manipulate the DOM. They can be structural or attribute-based:

  • Structural Directives: `*ngIf`, `*ngFor` to change the structure of the DOM.
  • Attribute Directives: `ngClass`, `ngStyle` to change appearance or behavior.
